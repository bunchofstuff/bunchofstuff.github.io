Lecture Pods

Week 1

  • Types of Apps

    • Web App

      • Makes use of web technologies and responsive web design

      • Pros

        • Quick Option for making existing content available via a mobile device
        • Only needs to be built once and would be usable for every device

      • Cons

        • UX inferior (compared to native apps)
        • Performance Inferior (compared to native apps)

    • Hybrid App

      • Combines elements of both web and native apps

      • Pros

        • bridge gaps between native apps and a mobile website

      • Cons

        • No unique mobile functions

      • Not only that, but, it's basically the same as a web app. It encourages lazy developing and will often have worse of performance than a native app

    • Native

      • Runs directly on a mobile device.

      • Pros

        • UI and UX smooth and engagin
        • Faster load times and better overall performance
        • Better discoverability and brand image

      • Cons

        • Require development for each platform
        • Expensive
        • Time Consuming

  • More than half, 55% of people in an oracle study say that a poor app or user experience would put them off buying or using a companies products or services.

      • What your app does and how it does it

      • Low Fidelity mockup
      • Either part of spec or built from spec

    • Prototype

      • Higher fidelity mockup
      • Using software like inVision or Marvel

    • Visual design

      • The is the more traditional sense of design

    • Development

      • Can be alongside the design
      • Actually implementing the app
      • Coding the whole lot

    • Iterate

      • Rinse and Repeat

  • Blueprint of an App or Website

    • User Needs
    • Site Objectives
    • Strategy
    • Scope
    • Structure
    • Skeleton
    • Surface

  • Personas

    • Personas are used to identify and target your audience to give a better user experience

    • Personas can be made up of

      • Age
      • Sex
      • Occupation
      • Hobbies
      • Likes/Dislikes
      • Other details relevant to the product you are developing.

    • Artefact persona

      • Sort of a compliment to a user persona

      • Rather than focusing on the person using it it focuses on the personality of the product

      • Ask questions like

        • If the interface were a person, what would she or he be like?
        • How would you expect users to react when they first view the product?
        • How would you describe this product to a friend?
        • How is the product different from competitive products?
        • Which celebrity ( or car, movie, etc) is the product most like? Why ?

 

Week 6

  • What is a user scenario

    • User scenarios are the path a user would take to your product
    • A narrative describing foreseeable interactions of types of users
    • Provide the Who, What, When, Where, Why and How
    • Much like personas can be used to better divine a users needs or wants

Week 7

  • App UI patterns

  • Launch

    • Used to reinforce branding
    • Allows app to be loaded
    • Don't confuse with home screen

  • Tray

    • Tray/Drawer/Sidebar/Hamburger
    • Active button + Tray Area
    • Used to hide info off screen
    • Enhance the screen they are on

  • List

    • Opening second and third level pages
    • Opening or exploring details for content section
    • Enlarging image
    • Opening screen or options
    • Don’t overdo a good thing
    • Don’t skip a step
    • No dead ends ( make sure it leads to an interactive result )
